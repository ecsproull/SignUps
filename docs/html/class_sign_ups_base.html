Detailed Description

SignUpsBase is the base class for most other classes.

It contains the strings for accessing the database plus code that is used in multiple places. It is used on both the user and admin side of the code.

protected

Creates a form that displays the rolling sessions along with their attendees.

This is a long function and studying the code is the only way to fully understand it. It allows members to select slots and also remove themselves from a previously selected slot. Some of the signups, such as monitors are not allowed to remove themselves within 2 days of their duty time. By default the user can see the default number of days that the signup specifies. Usually 30 days. There is a drop down where 60 and 90 days can be selected. The dropdown also allows to see the past 30 days to look at the history.

There is a lot of logic intertwined with the HTML that is being generated which makes this a hard function to follow. Good luck.

Parameters
string$signup_nameThe class name.
array$attendeesThe list of attendees for the class.
int$signup_idThe ID of the class.
object$templateThe template for the rolling class.
array$template_itemsEach one describe a signup for that day.
string$user_groupThe group that is allowed to sign up.
bool$adminThis being accessed by an administrator.
string$secretUnique id for a member.
string$descriptionDescription of the signup.
mixed$template_items2The template items for template2. Used when a template is changed at a predetermined date.
mixed$rolling_daysOverrides the standard number of rolling days.
